# 《出口退税备案单证与物流轨迹双链核验项目方案》优化建议

> 编制日期：2026年5月
>
> 审阅视角：税务领域（出口退税与税收风险管理）专家
>
> 适用对象：项目方案修订定稿前的内部审阅与外部讲解

---

## 一、整体评价

### 1.1 方案优点

1. **政策切入点准确**。以国令第826号"信息获取权"为依托，紧扣财税2026年11号、国税2026年第5号《管理办法》，论证逻辑链条完整。
2. **业务洞察专业**。FCL/LCL 物流模式的差异化分析、整柜"端到端"与散货"单证链"的核查策略区分，体现了对外贸物流实务的深度理解。
3. **创新点明确**。从"单笔轨迹验证"升级到"供应商维度集群分析"，对识别"买单配票"团伙化骗税具有突破性意义。
4. **闭环意识初步具备**。提出"预警—核查—验证"管理流程，并区分内外网展示策略，符合税务执法实际。

### 1.2 主要不足

| 维度 | 主要问题 |
|---|---|
| 政策依据 | 仅覆盖出口退税专项法规，缺《数据安全法》《个人信息保护法》《关税法》以及部委协作机制等支撑 |
| 概念表达 | "票货分离""三流合一""买单配票"等口语化表述较多，未形成法言法语严谨口径 |
| 风险维度 | 仅以物流空间轨迹为核心，缺资金流、主体关系、商品价格、历史信用等多维联动 |
| 集群分析 | 起运地热力图单一指标判定离散度，存在"真实多工厂供应商被误判"和"统一物流园集货被遗漏"两类漏洞 |
| 证据效力 | 第三方物流平台数据的取证规范、行政执法证据效力、企业异议救济机制未明确 |
| 流程深度 | 仅到"核查—验证"，未覆盖"处置—跟踪—反哺—行业治理"的完整治理链 |
| 服务平衡 | 偏重"打击骗税"，对诚信企业的"绿色通道""信用激励"着墨不足 |
| 业态适配 | 未提及跨境电商（9610/9710/9810）、市场采购（1039）、外贸综合服务等新业态的差异化方案 |
| KPI/试点 | 缺少分阶段目标、关键绩效指标、试点路径、项目风险与应对 |
| 文案风格 | "活水""试金石""双重保险"等修辞偏多；技术词（API/GIS）面向业务读者宜淡化 |

## 二、文案与结构优化

### 2.1 全文修辞建议

- **去口语化**：将"活水""试金石""双重保险""走向"等修辞替换为中性管理用语。例：
  - "源头'活水'" → "源头数据资源"
  - "检验监管能力的'试金石'" → "衡量监管效能的关键指标"
  - "双重保险" → "双重交叉验证"
- **去技术化**：面向税务管理读者，淡化 API、GIS、热力图、API 接口等技术词，改用业务化表达。例：
  - "对接第三方物流信息平台 API 接口" → "与第三方物流信息平台建立标准化数据通道"
  - "GIS 地图可视化" → "地理信息可视化展示"
- **法言法语**：
  - "票货分离" → 在首次出现处加注："货物流与单证流不一致的虚假出口业务（俗称'票货分离'）"
  - "买单配票" → "无真实出口业务的企业购买他人真实出口报关单据，配以本企业进项发票申报骗取退税（俗称'买单配票'）"
  - "三流合一" → "货物流、单证流、资金流'三流一致'"，并建议升级为"四流一致"（增加信息流）

### 2.2 章节结构调整建议

现行五段（背景—目标—架构—功能—成效）建议重构为七段，使逻辑更完整：

```
一、项目背景与建设必要性
二、指导思想与基本原则       ← 新增
三、建设目标与总体设计
四、核心业务能力             ← 替换"核心业务功能"
五、配套机制与制度保障       ← 新增（数据安全、证据规则、企业权益）
六、实施路径与阶段计划       ← 新增（试点—推广—迭代）
七、预期成效与评估指标       ← 增补 KPI
附录：术语表、典型骗税情形清单、政策依据清单
```

### 2.3 关键表格改写建议

- **整柜/散货对比表**：在末列"核查启示"基础上增加"易伪造点"列，便于审核人员快速识别造假手法。
- **数据来源表**：在"用途"列后增加"数据敏感级别""留存期限""调阅审批层级"三列，体现数据安全管控。
- **物流要素填写规范表**：增加"必填/选填""校验规则"列。

## 三、政策与法律依据扩展（第一章修订要点）

### 3.1 增补法律法规层级

建议形成"上位法—行政法规—部门规章—规范性文件—国际规则"五层政策矩阵：

| 层级 | 现有方案已引 | 建议增补 |
|---|---|---|
| 法律 | — | 《税收征收管理法》《增值税法》《关税法》（2024年12月施行）《数据安全法》《个人信息保护法》《行政处罚法》《行政强制法》 |
| 行政法规 | 国令826号 | 《税收征收管理法实施细则》《海关稽查条例》《外汇管理条例》 |
| 部门规章 | 财税2026年11号、国税2026年第5号 | 税务总局《税收风险管理工作指引》、海关《AEO 认证管理办法》、外汇局《货物贸易外汇管理指引》 |
| 规范性文件 | — | 中办国办《关于进一步深化税收征管改革的意见》（2021年）、税务总局"金税四期"建设方案、智慧税务建设规划 |
| 国际规则 | — | WCO《修订京都公约》、AEO 国际互认安排、《WTO 海关估价协定》、OECD 税收情报交换标准 |

### 3.2 数据合规专项依据

针对调取第三方物流轨迹（涉及个人信息——驾驶员，与商业秘密——承运企业）的合规边界，建议明确：

- **取数授权**：依据国令826号"税务机关可以依法获取……物流……信息"行使权力，但应同时遵守《数据安全法》第32条"必要原则"、《个人信息保护法》第13条"履行法定职责所必需"。
- **数据最小化**：仅获取出口业务关联时段、关联车辆/船舶的轨迹数据，不得扩大范围。
- **告知义务**：在企业向税务机关报送车牌号等信息时，应同步告知"该信息将用于物流轨迹调阅核验"。
- **跨境数据**：国际海运轨迹涉及境外承运人数据时，应符合《数据出境安全评估办法》及《个人信息出境标准合同办法》要求。

## 四、概念与术语严谨化

建议在方案前置"术语表"或在首次出现处加脚注，规范以下高频术语：

| 术语 | 严谨定义建议 |
|---|---|
| 备案单证 | 依据《管理办法》第四十五条规定，出口企业应按出口业务保留以备税务机关核查的单证，包括购销合同、出口货物运输单据、出口收汇单据等 |
| 票货分离 | 报关申报载明的货物与实际出口的货物不一致，或申报的货物根本不存在 |
| 买单配票 | 无真实自营出口业务的企业，购买他人真实出口报关单后，匹配本企业进项发票办理退税；或外贸公司以他人货物报关后，匹配第三方供货发票办理退税 |
| 配单出口 | 与"买单配票"近似但更广义，泛指报关单与发票、合同人为拼凑的虚假出口 |
| 三流一致 | 货物流（实际物流）、单证流（合同/报关/发票/单据）、资金流（货款/收汇）三者主体、金额、时间、内容相互对应 |
| 四流一致 | 在"三流"基础上增加信息流（订单、订舱、舱位确认、入仓通知、申报记录等），形成更全面的真实性证据链（建议本方案采用） |
| 物流轨迹 | 货物自起运地至最终目的港全过程的时间、地点、状态等可追溯记录，本方案重点为国内段公路运输轨迹与国际段海运轨迹 |
| 集群风险 | 多笔出口业务在主体、时空、商品、价格、单证等维度呈现共性异常，疑似团伙化、专业化骗税行为 |

## 五、风险识别维度补强（第二、三章修订要点）

### 5.1 由"单维"升级为"五维联动"

物流轨迹仅是一个维度。建议构建"主体—商品—资金—物流—信息"五维风险识别框架：

```
       主体维度
        ／  ＼
   信息维度    商品维度
       │  ✕  │      ← 五维交叉验证
   物流维度    资金维度
        ＼  ／
       业务真实性
```

| 维度 | 关键风险信号 |
|---|---|
| 主体 | 法定代表人/办税员同时关联多家高风险出口企业；股权穿透发现关联交易；企业成立时间短即出现大额退税；社保人数与出口规模严重不匹配 |
| 商品 | HS 编码与运输车辆吨位不匹配；价格离群（与同行业、同期、同口岸均价偏差超阈值）；"两高一资"或敏感品类；商品规格与装柜数量物理上不可能 |
| 资金 | 收汇路径异常（多家境外付款人指向同一账户）；资金回流（货款收回后短期内向供应商上游或关联自然人转出）；地下钱庄特征（拆分入账、短时进出） |
| 物流 | 国内段轨迹与申报起运地偏离；国际段船名航次与申报不符；同一车辆同时段被多家无关联企业申报；停留点异常；轨迹时间逻辑反序 |
| 信息 | 申报 IP、设备、办税终端集中；申报修改频次异常；电子合同、订舱信息与申报数据时序冲突 |

### 5.2 增加时序逻辑校验

物流轨迹除空间维度外，应强化时序校验，例如：

- 起运日期 → 进港日期 → 报关日期 → 离港日期 → 提单签发日期 → 到港日期之间的合理性
- 集装箱铅封时间晚于工厂装柜时间，但早于进港时间
- 同一车牌同一日内的起止地点是否物理可达（运输距离与时长是否合理）

## 六、"买单配票"集群分析模型升级（第三章第三节修订要点）

### 6.1 现有模型的两类典型漏洞

| 误判类型 | 场景 | 后果 |
|---|---|---|
| 假阳性 | 真实大型供应商有多个生产基地或合作工厂（如服装代工、电子组装），起运地天然分散 | 诚信企业被误纳入风险池，影响营商环境 |
| 假阴性 | 拼缝团伙将所有"买单"业务都从同一货代仓库前置集货后再发运，起运地高度聚集 | 模型失灵，无法识别真正的团伙骗税 |

### 6.2 升级建议：多指标集群研判

建议将单一"起运地离散度"升级为以下复合判定模型：

1. **起运地与登记地匹配度**
   - 计算起运地经纬度与供应商工商登记地、纳税申报地的距离
   - 设定合理半径阈值（如 50km），多笔业务平均偏离度高于阈值触发预警

2. **运力—产能匹配度**
   - 供应商社保人数、用电量、申报销售额作为产能基准
   - 出口业务月度货量超出产能基准 N 倍时触发预警

3. **车辆/船舶共享度**
   - 同一车牌在短时间内为多家无股权关联的供应商运输
   - 同一货代代理多家高风险企业，且这些企业彼此间存在共同的法人/办税员/账户

4. **集货前置点识别**
   - 提取多笔业务的实际装货地，识别"集货中心"（高频共用物流仓）
   - 集货中心若不在任一供应商的登记地附近，则需进一步核实是否为真实拼箱仓

5. **资金回流图谱**
   - 结合外汇局、银行授权数据，构建"境外付款人 → 出口企业 → 供应商 → 上游 → 自然人"的资金链路
   - 识别短期闭环（资金 T+N 内回流到初始付款人关联账户）

6. **关联交易穿透**
   - 通过工商、税务、银行多源数据建立企业关联图谱
   - 出口企业、供应商、货代、报关行间的隐性关联（共同股东、办公地址、电话、IP）

### 6.3 从"模型识别"到"案件画像"

建议在系统中沉淀"骗税情形特征库"，将每一类骗税手法的多维特征结构化（手法描述、关键信号、典型案例、识别规则），用于：

- 新预警业务自动比对相似案件
- 一线税务人员参考研判
- 模型迭代优化与新型手法学习

## 七、物流数据合规与证据效力（建议新增配套机制章节）

### 7.1 第三方数据的证据规则

| 证据要素 | 建议规则 |
|---|---|
| 合法性 | 数据来源平台需具备合法资质（如中交兴路具备道路货运信息服务经营许可），数据采集需符合驾驶员/承运人协议授权 |
| 真实性 | 与北斗/GPS 原始数据一致性核验；建立数据时间戳、哈希校验，防止中间篡改 |
| 关联性 | 调取数据须明确关联到本次出口业务的报关单号、车牌号、提单号 |
| 完整性 | 一次调取覆盖完整运输周期，禁止"剪辑式"片段取证 |
| 可追溯 | 数据调阅留痕：操作人、时间、原因、范围全程留痕，纳入审计 |

### 7.2 单一证据规则

明确**禁止仅依据物流轨迹一项证据定性骗税**。物流数据应作为辅助证据，与备案单证、资金流、申报数据、企业陈述等共同构成证据链。

### 7.3 企业异议与救济

- 企业对调取数据有知情权；对核查结论有陈述、申辩、复核权
- 设立"轨迹异议"通道：企业可对调取数据的真实性、完整性、关联性提出异议并提交反证
- 与现行《税收征管法》《行政处罚法》中的告知—听证—复议—诉讼救济链条衔接

## 八、流程闭环与处置升级（第三章修订要点）

将现行"预警—核查—验证"扩展为完整的**"七环治理闭环"**：

```
①风险扫描（事前/事中）→ ②申报预警（事中拦截）→
③信息报送（企业自证）→ ④审核研判（人机结合）→
⑤分级处置（暂缓退税/约谈/稽查/移送）→
⑥反馈跟踪（处置结果回流模型）→
⑦行业治理（典型案例库、政策辅导、合规共建）
```

### 分级处置建议

| 处置层级 | 适用情形 | 处置措施 |
|---|---|---|
| 一级 | 单证瑕疵、轻微逻辑不符 | 政策辅导、补正资料 |
| 二级 | 多项疑点但企业可合理解释 | 风险约谈、加强后续监管 |
| 三级 | 核心疑点无法澄清 | 暂缓办理退税、移交评估 |
| 四级 | 涉嫌骗税 | 移交稽查立案 |
| 五级 | 涉嫌犯罪 | 移送公安经侦 |

### 信用分级管控

与税务总局《纳税信用管理办法》、出口退税企业分类管理（一类至四类）衔接：

- 一类企业（A 级）：纳入"绿色通道"，物流轨迹核验抽查比例降低，退税审批提速
- 二类企业（B 级）：常规核验
- 三类企业（C 级）：加强核验，必要时全量物流轨迹比对
- 四类/M 级：全量、多维深度核查，必要时联合稽查

## 九、智能化能力建设（建议作为方案"二期"或与一期并行的"加强项"）

### 9.1 备案单证智能解析

- OCR + 版式识别：合同、装箱单、提单、报关单、运输单据自动结构化
- NLP 关键字段抽取：收发货人、唛头、品名、数量、金额、签发日期等
- 多语种支持：英、日、韩、阿等高频外贸语种

### 9.2 跨单证一致性自动比对

构建报关单—合同—发票—装箱单—提单—运单—收汇水单七联校验矩阵，自动标识不一致字段并量化偏差。

### 9.3 模型驱动的风险识别

- 监督学习：以历史已确认骗税案件为训练集，构建分类模型
- 无监督学习：基于行为特征聚类发现新型异常
- 知识图谱：企业—自然人—车辆—账户—地址—设备的关联图谱，支持穿透式研判

### 9.4 人机协同决策

- 系统输出"风险评分 + 关键证据 + 建议处置层级"
- 税务人员可采纳、否决或补充，反馈用于模型再训练
- 重大处置仍由税务人员最终决策，避免"算法黑箱"问责风险

## 十、部门协同与数据共享深化（建议新增配套机制章节）

### 10.1 跨部门数据共享矩阵

| 协同部门 | 共享数据 | 协同价值 |
|---|---|---|
| 海关 | 报关单、舱单、机检图像、AEO 信用、申报修改记录 | 物流与申报源头核验 |
| 外汇管理局 | 跨境收支、出口收汇核销、贸易外汇监测 | 资金流真实性核验 |
| 商务部门 | 对外贸易经营者备案、出口许可、跨境电商主体 | 主体合规与业态识别 |
| 公安经侦 | 涉案嫌疑主体、立案信息 | 重大风险联动 |
| 市场监管 | 工商登记、股权穿透、关联企业 | 主体关系穿透 |
| 银行/支付机构 | 账户信息、流水（依法授权） | 资金链路核验 |
| 交通运输部门 | 道路货运车辆、驾驶员、AIS 船舶轨迹 | 物流真实性核验 |
| 税务总局国际司 | 跨境税收情报、CRS 数据 | 跨境合规核查 |

### 10.2 协同机制建议

- 建立"出口退税真实性核验跨部门数据共享备忘录"
- 依托电子口岸、单一窗口、政务数据共享平台落地
- 设置数据使用边界：本项目数据仅用于出口退税真实性核验，禁止外溢

## 十一、治理理念与服务延伸

### 11.1 平衡"严打"与"善治"

现方案以打击为主基调，建议增加服务面向的表达：

- **激励诚信**：A 级企业绿色通道、退税即报即退；典型诚信案例宣传
- **辅导引导**：行前/行中政策辅导、骗税情形警示提醒、合规自查清单
- **行业共建**：与外贸协会、货代协会、报关协会共建合规标准与"白名单"
- **预防为主**：通过新办企业辅导、风险一对一提示，减少"事后追缴"

### 11.2 服务"双循环"与稳外贸大局

在"预期成效"章节增加宏观治理表述：

- 维护公平税收秩序，保护诚信企业利益
- 服务"稳外贸、稳就业、稳预期"国家战略
- 助力高水平对外开放与营商环境建设
- 提升我国海关 AEO 互认与税收征管国际声誉

## 十二、新业态适配（建议新增章节）

现方案默认传统货物贸易（0110/0130），未涵盖新业态。建议增补：

### 12.1 跨境电商出口（9610/9710/9810）

- **9610（B2C 直邮）**：单票小额、品类杂、无传统单证；建议依托海关跨境电商通关平台数据，按订单—物流—支付—申报四要素自动比对
- **9710（B2B 直接出口）/9810（出口海外仓）**：海外仓销售确认时点延后；建议建立"海外仓在库存货"专项核验

### 12.2 市场采购贸易（1039/1039A）

- 多品种、多供应商、低值；建议以"集货市场—集装箱"维度聚合分析，弱化逐票单证比对

### 12.3 外贸综合服务企业代办退税

- 责任主体复杂（生产企业 vs 综服企业）；建议建立综服企业风险监测专项指标

### 12.4 离岸贸易

- 货物不入境，依托香港、新加坡等离岸结算；建议结合外汇局跨境收支与海外仓数据综合判定

## 十四、附录建议

### 14.1 术语表

将第四章建议的术语统一收录于附录术语表，全文首次出现处加注。

### 14.2 政策依据清单

按"法律 → 行政法规 → 部门规章 → 规范性文件 → 国际规则"五层级整理。

### 14.3 典型骗税情形清单

参照税务总局历年发布的稽查案例汇编，结构化整理"买单配票""低值高报""循环出口""虚假出口""假自营真代理""国内空转出口"等典型情形，每种情形包括：手法描述、关键信号、识别规则、典型案例（脱敏）。

### 14.4 备案单证完整性核查清单

按《管理办法》第四十五条要求，结合本项目核查需要，形成"必备 + 建议 + 选用"三档单证清单，作为核查工作底稿模板。
